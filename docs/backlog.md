## BL-005 — Расширение API шаблонов в export-service

### Текущее состояние

Реализованы базовые эндпоинты: список (`GET /api/templates`), скачивание (`GET /api/templates/{type}/{filename}`), замена (`PUT /api/templates/{type}/{filename}`). Фронтенд (страница `/settings/templates`) уже поддерживает список, скачивание и замену.

### Необходимые эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/templates/{type}` | Загрузка нового шаблона (multipart, поле `file`). Имя файла берётся из upload. Возвращает 409 если файл уже существует |
| `DELETE` | `/api/templates/{type}/{filename}` | Удаление шаблона. Возвращает 404 если не найден |
| `POST` | `/api/templates/{type}/{filename}/copy` | Копирование шаблона в другой тип отчёта. Body: `{ "target_type": "ios" }`. Возвращает 409 если в целевом типе файл уже есть |
| `PUT` | `/api/templates/{type}/{filename}/rename` | Переименование файла. Body: `{ "new_name": "..." }`. Возвращает 409 при конфликте имён |

### Валидация

- Проверять что загруженный файл — валидный DOCX (попробовать открыть через `python-docx`, вернуть 422 при ошибке)
- Ограничить размер файла (10 MB max)
- Sanitize имени файла (только `[a-zA-Z0-9_.-]`)

### Доработки фронтенда (после реализации бэкенда)

- Кнопка "Добавить шаблон" на вкладке типа отчёта
- Кнопка "Удалить" в строке шаблона (с подтверждением)
- Контекстное меню или кнопка "Копировать в..." для копирования между типами
- Drag-and-drop загрузка файлов на область вкладки
